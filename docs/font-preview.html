Font Comparison

Each card uses one UI font + one monospaced font for code/math-style text.

Source Sans 3 + Source Code Pro

UI + body: Source Sans 3 · Code: Source Code Pro

The quick brown fox jumps over the lazy dog. In LaTeX, \(E=mc^2\) and \(\sum_{i=1}^{n} i\) should read cleanly.

Toolbar, menus, and editor controls look clean and modern.

\begin{equation}\n \frac{\partial u}{\partial t} = \alpha \nabla^2 u\n\end{equation}

IBM Plex Sans + IBM Plex Mono

UI + body: IBM Plex Sans · Code: IBM Plex Mono

The interface feels neutral, technical, and highly readable with subtle character width.

Great for research notes, references, and frequent command entry.

\section{Methods}\n\label{sec:methods}\n\newcommand{\vect}[1]{\boldsymbol{#1}}

Nunito Sans + JetBrains Mono

UI + body: Nunito Sans · Code: JetBrains Mono

A friendlier, less rigid UI while keeping math and code lines crisp.

This pairing often feels warm without losing professionalism.

\documentclass{article}\n\usepackage{amsmath}\n\title{Experiment Notes}

Inter + Fira Code

UI + body: Inter · Code: Fira Code

Inter gives excellent UI density and Fira Code adds readable code with optional ligatures.

If you use keyboard-heavy workflows, this is a practical default.

for i in {1..3}; do\n echo \"Run $i\"\ndone

Merriweather + JetBrains Mono

UI + body: Merriweather · Code: JetBrains Mono

Merriweather brings a more editorial style suitable for long-form notes and writing blocks.

This can look sharp for prose-heavy areas and still keep code distinct.

\begin{tikzpicture}\n\draw[->] (0,0) -- (2,2);\n\end{tikzpicture}
